Packet Tracer ile Ospf Yapılandırması

Temmuz 4, 2018 / by Erdem Yıldız

Merhaba arkadaşlar bugün sizlere Ospf Dinamic Routing hakkında açıklama yapıp daha sonra Packet Tracer üzerinde bir uygulama ile kullanımını göstereceğim. Haydi bir giriş yapalım...

OSPF

(Open Shortest Path First - İlk Açık Yöne Öncelik) Link-state (Hat Durumu) protokolü olarak tasarlanmıştır. Link-state yönlendirme protokolleri, topolojinin tamamını görebildiği gibi, ağ değişikliklerinde Triggered update (Tetiklenmiş Güncelleme) gönderir. Buna göre yönlendiriciler ağdaki iki nokta arasında bulunan tüm yolların bilgisine ulaştıktan sonra SPF (Shortest Path First - Önce En Kısa Yol) algoritmalarını kullanarak hangi yolun en iyisi olduğuna karar verirler. Ayrıca Link-state Refresh (Hat Durumu Güncellemesi) olarak bilinen, 30 dakikada bir periyodik güncellemeler gönderir.

  • VLSM desteği vardır
  • Sınıfsız bir yapıya sahiptir
  • AD değeri 110'dur.
  • OSPF protokolünde Area (Alan) değeri önemlidir. Geniş ağlarda alanlara ayırarak yönlendirme tablosunun büyüklüğü ve ağın karmaşıklığı azaltılabilir
  • Metrik hesaplamasında Cost adı verilen bir hesaplama kullanır.Ospf için en iyi yol en düşük maliyetli yoldur.

Daha fazlası için tıklayın 

                                            #OSPF Dinamic Routing

                                            #I.adım
                                            router ospf 1-65535 Process Id

                                            #II.adım (Router'ın önündeki ve arkasında network bloklarını adedi kadar yazın)
                                            network ip-blok - Wild card area 0-4294967295

Yukarıdaki fotoğraf'ta 3 adet router ve onlara bağlı 3 adet switch ve bu cihazlara bağlı 2'şer adet host görüyoruz. Her switch'deki host'ların ip blokları birbirinden farklıdır.Biz bunların haberleşmesi için Dinamic Routing protokollerinden olan Ospf'i kullancağız.

  • Host A ip adresi -> 192.168.1.2 , gateway adresi(R1 gig0/0) -> 192.168.1.1
  • Host B ip adresi -> 192.168.1.3 , gateway adresi(R1 gig0/0) > 192.168.1.1
  • R1(gig0/0) ip addresi -> 192.168.1.1 (Gateway)
  • R1(se0/1/1) ip addresi -> 192.168.100.1
  • R1(se0/1/0) ip addresi -> 192.168.200.1
  • Host C ip adresi -> 192.168.2.2 , gateway adresi(R2 gig0/0) -> 192.168.2.1
  • Host D ip adresi -> 192.168.2.3 , gateway adresi(R2 gig0/0) -> 192.168.2.1
  • R2(gig0/0) ip addresi -> 192.168.2.1 (Gateway)
  • R2(se0/1/1) ip addresi -> 192.168.250.2
  • R2(se0/1/0) ip addresi -> 192.168.200.2
  • Host E ip adresi -> 192.168.3.2 , gateway adresi(R2 gig0/0) -> 192.168.3.1
  • Host F ip adresi -> 192.168.3.3 , gateway adresi(R2 gig0/0) -> 192.168.3.1
  • R3(gig0/0) ip addresi -> 192.168.3.1 (Gateway)
  • R3(se0/1/1) ip addresi -> 192.168.100.2
  • R3(se0/1/0) ip addresi -> 192.168.250.1

Yukarıda şekilde'ki router'ların routing table örneğini göstermek istedim.Başında "O" harfi olan'lar Osfp'i temsil etmektedir.Kurallara biraz daha bakarsak hangi ip bloğuna hangi bacak'tan erişebileceğimizi görebiliriz

Örnek Ospf Dinamic Routing Yapılandırması

                                            #R1 için Ospf
                                            router ospf 10
                                            network 192.168.100.0 0.0.0.255 area 0
                                            network 192.168.200.0 0.0.0.255 area 0
                                            network 192.168.1.0 0.0.0.255 area 0

Yukardaki kuralın benim için anlamı şudur.İlk başta kendimizce bir numara verip Ospf'i tanımlıyoruz Daha sonra R1'in 3 adet bacağı aktif olduğunu biliyoruz.Biz bu bacakları diğer router'lar görmesi için kullanıma açıyoruz.
